## 2. 설치 방법

### 방법 A — 자동 설치 스크립트 (권장)

```bash
# 소스 다운로드
git clone https://github.com/22blabs/govbot-kr.git
cd govbot-kr

# 설치 스크립트 실행
chmod +x install.sh
./install.sh
```

스크립트가 자동으로 다음 작업을 수행합니다:
- Docker 설치 확인
- `.env` 파일 생성 (SECRET_KEY 자동 생성)
- 서비스 빌드 및 시작
- 데이터베이스 마이그레이션

### 방법 B — 수동 설치

```bash
# 1. .env 파일 생성 (.env.example 복사)
cp .env.example .env
# .env 파일을 편집하여 설정 입력

# 2. 서비스 시작
docker compose up -d

# 3. DB 마이그레이션
docker compose exec backend alembic upgrade head
```

## 3. 초기 설정

### 3-1. 관리자 계정 생성

```bash
docker compose exec backend python -m app.scripts.create_admin
```

프롬프트에 따라 입력:
- **테넌트 ID**: 기관 식별자 (영문 소문자, 예: `dongducheon`)
- **이메일**: 관리자 이메일 주소
- **비밀번호**: 8자 이상

### 3-2. 기관 정보 설정

`.env` 파일에서 아래 항목을 설정하거나, 추후 API로 TenantConfig를 등록하세요.

| 항목 | 설명 | 예시 |
|------|------|------|
| `tenant_name` | 기관명 | 동두천시 |
| `phone_number` | 대표 전화번호 | 031-860-2000 |
| `fallback_dept` | 폴백 안내 부서 | 민원과 |

### 3-3. 대시보드 접속

서비스 시작 후 브라우저에서 접속:

- **대시보드**: http://서버IP:3000
- **API 문서**: http://서버IP:8000/docs

## 4. 대시보드 사용법

로그인 화면에서 **테넌트 ID**, **이메일**, **비밀번호**를 입력합니다.

### 메뉴 구성

| 메뉴 | 설명 |

| 💬 시뮬레이터 | 실제 민원 응답 테스트 |

## 5. FAQ 관리

FAQ는 **Tier A** 응답에 사용됩니다. 유사도 ≥ 0.85 이상이면 즉시 답변합니다.

### FAQ 추가

1. **FAQ 관리** 메뉴 클릭
2. **+ FAQ 추가** 버튼 클릭
3. 카테고리, 질문, 답변 입력
4. **저장** 클릭

### 효과적인 FAQ 작성 방법

- **질문**: 시민이 실제로 사용하는 표현으로 작성 (예: "주민등록등본 어디서 발급해요?")
- **답변**: 명확하고 완결된 문장으로 작성 (300자 이내 권장)
- **카테고리**: 관련 업무 분류 (예: 민원, 세금, 복지)

> 동일한 의미의 질문을 다르게 표현한 FAQ를 여러 개 등록하면 인식률이 높아집니다.

## 6. 문서 관리 (RAG)

문서는 **Tier B/C** 응답에 사용됩니다. 문서 내용을 기반으로 답변을 생성합니다.

### 지원 파일 형식

| 형식 | 확장자 | 비고 |
|------|--------|------|
| 텍스트 | `.txt` | 권장 |
| PDF | `.pdf` | 이미지 PDF 불가 |
| Word | `.docx` | |
| 마크다운 | `.md` | |

### 문서 업로드 절차

1. **문서 관리** 메뉴 클릭
2. **+ 문서 업로드** 버튼 클릭
3. 파일 선택 (최대 50MB)
4. 처리 완료 후 상태가 `processed`로 변경
5. **승인** 버튼 클릭 → 활성화 (`is_active: true`)

> **주의**: 승인하지 않으면 검색에 사용되지 않습니다. 반드시 승인 후 시뮬레이터로 테스트하세요.

### 문서 처리 상태

| 상태 | 설명 |
|------|------|
| `pending` | 처리 대기 중 |
| `processed` | 처리 완료, 승인 대기 |
| `parse_failed` | 파일 파싱 오류 (다른 형식으로 변환 후 재업로드) |
| `embedding_unavailable` | 임베딩 모델 오류 |

## 7. 민원 이력 조회

처리된 모든 민원 기록을 조회할 수 있습니다.

- **발화 내용**: 개인정보 마스킹 처리 후 표시
- **사용자 키**: SHA-256 해시값만 표시 (원본 미저장)
- **Tier 필터**: A/B/C/D 별 조회 가능

> 개인정보 보호법에 따라 원문은 시스템에 저장되지 않습니다.

## 8. 악성 감지 · 이용 제한

자동으로 감지된 악성 사용자를 관리합니다.

### 제한 레벨

| 레벨 | 이름 | 설명 |
|------|------|------|
| 0 | NORMAL | 정상 |
| 1 | WARNED | 경고 |
| 2 | THROTTLED | 응답 지연 |
| 3 | SUSPENDED | 일시 중지 (편집자 수동 해제) |
| 4 | BLOCKED | 차단 (관리자만 해제) |
| 5 | BANNED | 영구 차단 |

### 수동 조치

- **단계 상향**: 현재 레벨에서 1단계 상향 (최대 SUSPENDED=3)
- **해제**: 제한 해제, NORMAL(0)로 복귀

> Level 4(BLOCKED) 이상은 관리자만 해제할 수 있습니다.

## 9. 웹 채팅 위젯 삽입

지자체 홈페이지에 채팅 위젯을 삽입하려면 `</body>` 태그 앞에 아래 코드를 추가합니다.

```html
<script
  src="http://서버IP:3000/widget/govbot-widget.js"
  data-tenant="your-tenant-slug"
  data-api="http://서버IP:8000"
  data-title="AI 민원 도우미"
  data-color="#2563eb"
></script>
```

| 속성 | 설명 | 예시 |
|------|------|------|
| `data-tenant` | 테넌트 슬러그 | `dongducheon` |
| `data-api` | API 서버 주소 | `https://bot.dongducheon.go.kr` |
| `data-title` | 위젯 제목 | `AI 민원 도우미` |
| `data-color` | 테마 색상 (HEX) | `#2563eb` |

## 10. LLM 연동 설정

기본값은 LLM 없이 FAQ + RAG만 사용합니다. LLM을 연동하면 **Tier C** (문서 기반 자연어 답변)가 활성화됩니다.

`.env` 파일 수정:

### Anthropic Claude 사용

```env
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-api03-...
```

### OpenAI GPT 사용

```env
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-...
```

설정 후 서비스 재시작:

```bash
docker compose restart backend
```

> **비용 주의**: LLM API는 사용량에 따라 비용이 발생합니다. 시뮬레이터로 충분히 테스트 후 활성화하세요.

## 11. 백업 · 복구

### 데이터베이스 백업

```bash
# 백업
docker compose exec db pg_dump -U govbot govbot > backup_$(date +%Y%m%d).sql

# 복구
docker compose exec -T db psql -U govbot govbot < backup_20260101.sql
```

### 전체 볼륨 백업

```bash
# 모든 데이터 볼륨 백업
docker run --rm \
  -v govbot-kr_postgres_data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/postgres_$(date +%Y%m%d).tar.gz /data
```

### 백업 자동화 (crontab)

```bash
# crontab -e 에 추가 (매일 새벽 2시)
0 2 * * * cd /path/to/govbot-kr && docker compose exec db pg_dump -U govbot govbot > /backup/govbot_$(date +\%Y\%m\%d).sql
```

## 12. 문제 해결

### 서비스가 시작되지 않을 때

```bash
# 로그 확인
docker compose logs backend
docker compose logs db

# 서비스 재시작
docker compose restart
```

### 포트가 이미 사용 중일 때

```bash
# 사용 중인 프로세스 확인
sudo lsof -i :8000
sudo lsof -i :3000

# docker-compose.yml에서 포트 변경
# "8000:8000" → "8080:8000"
```

### 데이터베이스 연결 오류

```bash
# DB 컨테이너 상태 확인
docker compose ps db

# DB 직접 연결 테스트
docker compose exec db psql -U govbot -c "\l"

# 마이그레이션 재실행
docker compose exec backend alembic upgrade head
```

### 임베딩 모델 다운로드 실패

임베딩 모델(jhgan/ko-sroberta-multitask)은 첫 실행 시 자동 다운로드됩니다.

```bash
# 인터넷 연결 확인
docker compose exec backend curl -I https://huggingface.co

# 모델 캐시 초기화
docker compose exec backend rm -rf /root/.cache/huggingface
docker compose restart backend
```

### API 응답이 느릴 때

- 임베딩 모델 워밍업 완료 여부 확인: `/ready` 엔드포인트 응답 확인
- 서버 리소스 확인: `docker stats`

## 13. Windows WSL2 설치 가이드

Windows 11/10에서 GovBot KR을 실행하려면 WSL2를 사용합니다.

### 13-1. WSL2 설치

PowerShell을 **관리자 권한**으로 실행:

```powershell
# WSL2 설치 및 Ubuntu 설치
wsl --install

# 재시작 후 Ubuntu 사용자 설정
```

재시작 후 Ubuntu 터미널이 열리면 사용자 이름과 비밀번호를 설정합니다.

### 13-2. Docker Desktop 설치

1. [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/) 다운로드 및 설치
2. Settings → Resources → WSL Integration → Ubuntu 활성화
3. Docker Desktop 재시작

### 13-3. GovBot KR 설치

Ubuntu 터미널에서:

```bash
# Git 설치 (필요시)
sudo apt update && sudo apt install -y git

# 프로젝트 클론
git clone https://github.com/22blabs/govbot-kr.git
cd govbot-kr

# 설치 스크립트 실행
chmod +x install.sh
./install.sh
```

### 13-4. 브라우저 접속

WSL2는 localhost로 Windows 브라우저에서 바로 접근 가능합니다:
- 대시보드: http://localhost:3000
- API 문서: http://localhost:8000/docs

### 13-5. 서비스 자동 시작 설정

Windows 시작 시 GovBot KR을 자동으로 실행하려면:

1. Docker Desktop 설정 → General → **Start Docker Desktop when you log in** 체크
2. WSL2에서 `~/.bashrc` 또는 `~/.profile`에 추가:

```bash
# GovBot KR 자동 시작 (선택사항)
# cd ~/govbot-kr && docker compose up -d 2>/dev/null
```

> WSL2에서 파일을 Windows 경로(`/mnt/c/...`)에 저장하면 성능이 저하됩니다. WSL2 홈 디렉터리(`~/`)를 사용하세요.
